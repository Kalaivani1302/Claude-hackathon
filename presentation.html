Claude Primitives

All 5. No shortcuts.

Each primitive solves a real production failure mode.

🔒 System Prompt Isolation
Agent identity lives in the system prompt. Ticket bodies are user messages only. TKT-003 sends "IGNORE ALL PREVIOUS INSTRUCTIONS" — the agent detects it and refuses.
🔧 Tool Use — Anti-Hallucination Layer
7 mandatory tools in sequence. No fact is invented. User profiles, KB solutions, incident states — all come from tool results. Wrong tool output → wrong decision, not hallucinated decision.
👤 Human-in-the-Loop
Agent pauses mid-loop via Promise/Map pattern. SSE emits approval request. Browser shows modal. Human clicks Approve / Reject / Escalate. Agent resumes. VIP tickets, SECURITY tickets, injections: always NEEDS_HUMAN.
🧠 Chain-of-Thought Logging
System prompt requires REASONING: block before the JSON decision. Streamed to amber panel in UI. "Why did the agent route this to Security-Response?" — the audit trail answers it.
📐 Structured Output — Zod + Pydantic
TriageDecisionSchema (TypeScript/Zod) and TriageDecision (Python/Pydantic) validate every output. "IT Support" queue → rejected. "P5" priority → rejected. Invalid JSON → error event, not silent failure.

Technical Depth

Production patterns, not demo shortcuts

The HITL Promise/Map pattern

// Agent pauses here — waits for human case "request_human_approval": { return new Promise((resolve) => { emit("human_approval_required", input); pendingApprovals.set(input.ticket_id, resolve); }); } // POST /api/approve unblocks agent const resolve = pendingApprovals.get(ticket_id); resolve({ status: decision });

No polling. No WebSocket. No Redis. Native Promise/Map. The agent is suspended in the JavaScript event loop until the human clicks.

Structured output validation

const TriageDecisionSchema = z.object({ priority: z.enum(["P1","P2","P3","P4"]), decision: z.enum(["AUTO_RESOLVE","NEEDS_HUMAN", "ROUTE_TO_QUEUE"]), queue: z.enum(VALID_QUEUES), confidence:z.number().min(0).max(1), ... }); // Throws on "IT Support" queue or "P5" priority TriageDecisionSchema.parse(decision);

Invalid queue names and out-of-range confidence scores are caught before they hit the ITSM. Same schema in Pydantic for Python runtime.

Next Steps

What a production version looks like

The foundation is solid. Here's what goes on top.

Short term (1 week)

  • 🔌Real ITSM integration — ServiceNow REST API. Tool contract already defined, one-function swap.
  • 🔐Auth on /api/approve — JWT tied to approver identity. Audit trail needs a name.
  • ⏳Approval timeout — 5-minute SLA. Promise resolves with status: 'timeout' if no human responds.
  • 🧪Test harness — vitest, 20 synthetic tickets, mocked Anthropic client, decision shape assertions.

Medium term (1 month)

  • 🧠Persistent memory per user — "3rd VPN ticket this month → escalate to infra review"
  • 📊Confidence calibration evals — 100 synthetic tickets vs. human baseline. Measure precision/recall.
  • 🚦Request queue — p-queue with concurrency 5. No API quota exhaustion under load.
  • 🌊Streaming reasoning — word-by-word amber panel using stream: true + get_final_message().
